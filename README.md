## 1.top
top 명령어는 현재 **OS의 메모리 사용량**, **CPU 사용량** 등을 실시간으로 갱신되는 값으로 나타내주는 명령어로   
프로그램 구동시에 서버의 부하가 어느정도 되는지 테스트나 모니터링 할 때 주요하게 사용한다.   
top 명령어의 옵션은 다음과 같이 정리할 수 있다.
| 명령어 | 기능 | 문법 |
|:--------:|:-------|:-------|
| **-d** | 업데이트 간격(단위: 초) 지정 | `top -d` |
| **-p** | 특정 프로세스 ID만 모니터링 | `top -p PIDNUM` |
| **-n** | 실행 횟수 지정 | `top -n` |
<br/>
<br/>
<br/>

## 2.ps
ps 명령어는 "프로세스 상태"를 나타내며 시스템에서 현재 실행 중인 프로세스에 대한 정보를 제공하는 데 사용된다.   
이를 통해 사용자가 프로세스, 상태 및 이와 관련된 다양한 속성을 모니터링할 수 있고, 일반적으로 프로세스 관리, 문제 해결, 시스템 모니터링에 쓰인다.   
ps 명령어의 옵션은 다음과 같이 정리할 수 있다.
| 명령어 | 기능 | 문법 |
|:--------:|:-------|:-------|
| **-A** | 모든 프로세스 정보 표시 | `ps -A` |
| **-p** | 지정한 프로세스의 정보만 출력 | `ps -p 프로세스id` |
| **-u** | 사용자 이름과 작동 시간 표시 | `ps -u 사용자 이름` |
<br/>
<br/>
<br/>

## 3.jobs
jobs 명령어는 현재 쉘 세션에서 실행 중인 백그라운드 및 포그라운드 작업의 목록을 표시하는 명령어이다.   
이 명령어는 주로 작업 제어와 관련된 정보를 제공하며 이를 통해 사용자는 어떤 작업이 실행 중인지, 멈춰 있는지 등을 확인할 수 있다.   
jobs 명령어의 옵션은 다음과 같이 정리할 수 있다.
| 명령어 | 기능 | 문법 |
|:--------:|:-------|:-------|
| **-l** | 프로세스 ID까지 함께 표시 | `jobs -l` |
| **-p** | 작업 ID 대신 프로세스 ID만 출력 | `jobs -p` |
| **-r** | 실행 중인 작업만 출력 | `jobs -r` |
| **-s** | 중지된 작업만 출력 | `jobs -s` |
<br/>
<br/>
<br/>

## 4.kill
kill 명령어는 프로세스에 특정 시그널을 보내어 동작을 제어하는 명령어로 대부분 프로세스를 종료할 때 사용하지만,   
종료뿐 아니라 일시정지, 재개 등 다양한 신호 전송도 가능하다.   
기본적으로 kill 명령어를 사용할 때에 별도의 옵션을 주지 않는다면 시그널 15가 사용된다.   
kill 명령어의 옵션은 다음과 같이 정리할 수 있다.
| 명령어 | 기능 | 문법 |
|:--------:|:-------|:-------|
| **-l** | 시그널 목록 출력 | `kill -l` |
| **-s** | 보낼 시그널을 지정 | `kill -s ` |
<br/>

#### 또한 kill -l로 볼 수 있는 시그널의 목록 중 20번 까지의 내용은 다음과 같다.
| 번호 | 시그널 | 의미 |
|:----:|:-------:|:----|
| 1 | SIGHUP | 종료 |
| 2 | SIGINT | 인터럽트 종료 |
| 3 | SIGQUIT | 종료 후 코어 덤프 파일 생성 |
| 4 | SIGILL | 잘못된 명령어 신호 |
| 5 | SIGTRAP | 디버깅 목적으로 사용되는 신호 |
| 6 | SIGABRT | 중단 신호 |
| 7 | SIGBUS | 프로세스 버스 오류 |
| 8 | SIGFPE | 0으로 나눌 때 발생하여 종료되는 신호 |
| 9 | SIGKILL | 강제 종료 |
| 10 | SIGUSR1 | 사용자 정의 조건 신호 |
| 11 | SIGSEGV | 프로세스가 오작동할 때 커널에 의해 프로세스로 전송되는 신호 |
| 12 | SIGUSR2 | 사용자 정의 조건 신호 |
| 13 | SIGPIPE | 구문이 이상한 파이프에 대한 신호 |
| 14 | SIGALRM | real-time 또는 clock-time이 종료되면 전송되는 신호 |
| 15 | SIGTERM | 실행 중지를 프로세스에 요청하는 신호 |
| 16 | SIGSTKFLT | 스택 오류 신호 |
| 17 | SIGCHLD | 상위 프로세스가 하위 프로세스를 잃으면 상위 프로세스에 전송되는 신호 |
| 18 | SIGCONT | 프로세스가 SIGTSTP 또는 SIGSTOP 신호에 의해 중지된 후 계속 실행되게 하는 신호 |
| 19 | SIGSTOP | 프로세스 일시 중지, 프로세스가 신호 무시 불가 |
| 20 | SIGTSTP | 프로세스 일시 중지, 프로세스가 신호 무시 가능 |
<br/>

#### 자주 사용하는 시그널
- **9(SIGKILL)**: 강제 종료 (**데이터 손실 위험**)
- **15(SIGTERM)**: 정상 종료 요청 (기본 동작)
